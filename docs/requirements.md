# BlogViz AI — 要件定義書

**バージョン：** 1.1.0
**作成日：** 2026-03-21
**ステータス：** 確定

---

## 1. プロダクトビジョン

> ブログ本文を貼り付けるだけで、AIが「どのセクションに・何枚・どんな画像が必要か」を判断し、
> 解説用HTMLスライドを自動生成、Playwright MCPでスクリーンショット画像化して、
> 挿入箇所を指定して返してくれるツール

---

## 2. ターゲットユーザー

| ユーザー像 | 課題 |
|------------|------|
| ブロガー（個人・企業） | 文章だけの記事が読みにくい。でも画像を作る時間・スキルがない |
| テクニカルライター | 手順説明に図解が欲しいが、毎回作るのが大変 |
| コンテンツマーケター | 記事の理解度・滞在時間を上げたいが画像制作がボトルネック |

---

## 3. ユーザーストーリー

> 「私はブロガーとして、
> ブログ全文を貼り付けるだけで
> どこに・どんな解説画像を入れるべきか提案してほしい。
> そして実際の画像ファイルとして受け取りたい。」

---

## 4. 確定した設計方針（Q1〜Q4 回答）

| # | 質問 | 回答 | 決定内容 |
|---|------|------|----------|
| Q1 | デザインテーマの選択方法 | **C：ユーザー手動選択** | テーマ一覧をUIで表示し、ユーザーが生成前に選択する |
| Q2 | 確認ステップの有無 | **A：確認ステップあり** | 提案レポートを表示 → ユーザーが承認/削除 → 画像生成 |
| Q3 | 最終出力形式 | **A：PNG の ZIP のみ** | 画像ファイル群 + 挿入位置マップ JSON の ZIP |
| Q4 | コンテンツタイプ判定 | **A：AI 全自動** | AIが記事内容を読んで最適な表現タイプを自動選択 |

---

## 5. システムフロー（確定版）

```
STEP 1：ブログ全文テキストを入力エリアにペースト
         ↓
STEP 2：AI — テキスト構造解析
        ・セクション（見出し）分割
        ・各セクションの「内容タイプ」判定（AI全自動）
        ・全体の記事トーン・テーマ把握
        ・視覚化優先度スコアリング（0〜10）
         ↓
STEP 3：AI — 画像提案レポート生成
        ・必要な画像の総枚数
        ・各画像の挿入位置（セクション名・段落番号）
        ・各画像の内容・目的・表現タイプ（AI自動判定）
        ・優先度スコア（★1〜5）
         ↓
      ★ユーザー確認ステップ（Q2：確認ステップあり）
        ・提案カードを一覧表示
        ・不要な提案を削除
        ・「この内容で画像生成する」ボタンで確定
         ↓
STEP 4：ユーザーがデザインテーマを選択（Q1：手動選択）
        ・テーマ一覧（例：Tech / Minimal / Warm 等）
        ・全画像に同じテーマを適用
         ↓
STEP 5：AI — 各画像用HTMLスライドを生成
        ・1提案 = 1 HTMLファイル
        ・選択したテーマを適用
        ・図解・フロー・比較表・概念図等を表現
         ↓
STEP 6：playwright-core（Railway Docker上のChromium）— HTMLをスクリーンショット
        → PNG画像ファイルとして出力
        ※日本語フォント（Noto CJK + IPA）適用済み
         ↓
STEP 7：出力（Q3：PNG の ZIP のみ）
        ・image_01.png 〜 image_N.png
        ・insertion_map.json（挿入位置マップ）
        → ZIP でダウンロード
```

---

## 6. 機能要件

### F-01｜テキスト入力

| 項目 | 内容 |
|------|------|
| 入力形式 | プレーンテキスト / Markdown（どちらでも可） |
| 入力方法 | テキストエリアへの直接ペースト |
| 文字数制限 | 最大 20,000文字（MVP） |
| 言語 | 日本語・英語（両対応） |

### F-02｜テキスト構造解析（AIエンジン）

AIが以下を自動抽出する：

| 解析項目 | 説明 |
|----------|------|
| セクション分割 | H1〜H3 見出しまたは段落の意味的なまとまりで分割 |
| 内容タイプ判定 | 各セクションが「手順系・概念説明・比較・数値データ・事例紹介」のどれか自動判定 |
| 記事全体テーマ | 記事のカテゴリ・トーン（技術系・ライフスタイル・ビジネス等）を把握 |
| 視覚化優先度スコア | 「画像があると理解度が上がる度合い」を0〜10でスコアリング |

### F-03｜画像提案レポート生成

AIが以下を含む提案レポートをカード形式で出力（ユーザーが削除・確認可能）：

```
📊 総提案画像枚数：N枚

[画像カード #1]
  挿入位置 ：「セクション名」の冒頭/末尾
  理由     ：[なぜここに画像が必要か]
  表現タイプ：[AIが自動選択: フローチャート / 比較表 / ステップ図 / 概念図 / コード解説図]
  優先度   ：★★★★★（5段階）
  [削除ボタン] [このカードを保持]

[画像カード #2]
  ...

[「この内容で画像生成する」ボタン]
```

### F-04｜デザインテーマ選択（確定：ユーザー手動選択）

| 項目 | 仕様 |
|------|------|
| 選択タイミング | 提案確認後・HTML生成前 |
| 選択方法 | テーマカード一覧からクリック選択 |
| テーマ候補（MVP） | Tech（青系・モノスペース）/ Minimal（白黒・シンプル）/ Warm（オレンジ・丸み）|
| 適用範囲 | 全画像に同じテーマを一括適用 |

### F-05｜HTMLスライド生成

| 項目 | 仕様 |
|------|------|
| 出力 | 1提案 = 1 `.html` ファイル |
| サイズ | 幅 800px × 高さ 450px（16:9） |
| 表現タイプ | フロー図・比較表・番号付きステップ・概念図・コード解説（AIが自動選択） |
| 日本語フォント | `font-family: 'Noto Sans JP', 'IPAexGothic', sans-serif` |
| 文字エンコーディング | `<meta charset="UTF-8">` 明示 |
| テーマ適用 | ユーザー選択テーマの CSS を埋め込み |

### F-06｜playwright-core によるスクリーンショット

| 項目 | 仕様 |
|------|------|
| 実行方法 | `playwright-core` + Railway Docker上のシステムChromium で HTML を開いてスクリーンショット |
| 出力形式 | PNG |
| 解像度 | 2x（Retina 対応、実質 1600×900px） |
| 日本語フォント | Noto CJK + IPA フォントをDockerfileでapt-getインストール |
| 文字化け対策 | HTML に `@import Google Fonts Noto Sans JP` + ローカルフォントのフォールバック |

### F-07｜出力（確定：PNG の ZIP のみ）

| 出力物 | ファイル名 | 内容 |
|--------|-----------|------|
| 画像ファイル群 | `image_01.png` 〜 `image_N.png` | PNG スクリーンショット |
| 挿入位置マップ | `insertion_map.json` | セクション名・段落番号・挿入方向・表現タイプを記載 |
| ZIP ファイル | `blogviz_images.zip` | 上記をまとめてダウンロード |

`insertion_map.json` の構造例：
```json
{
  "version": "1.0",
  "items": [
    {
      "imageFilename": "image_01.png",
      "sectionHeading": "インストール手順",
      "insertPosition": "before",
      "paragraphIndex": 0,
      "visualType": "flowchart",
      "priority": 5
    }
  ]
}
```

## 7. 非機能要件

| 項目 | 目標値 |
|------|--------|
| レスポンス時間（提案レポートまで） | 10秒以内 |
| 画像1枚あたりの生成時間 | 5秒以内（playwright-core） |
| 日本語表示 | 文字化けゼロ（Noto CJK + IPA 保証） |
| 最大同時処理画像数 | 10枚/リクエスト（MVP） |
| 対応ブラウザ（UI） | Chrome 最新 / Safari 最新 |

---

## 8. MVP スコープ（最小構成）

MVP として最初にリリースする機能：

1. テキスト入力エリア（最大 20,000文字）
2. AI解析 → 画像提案レポートをカード形式で表示
3. ユーザーによる提案カード削除・確認ステップ
4. デザインテーマ選択 UI（3テーマ）
5. HTML スライド自動生成
6. playwright-core で PNG 化
7. ZIP ダウンロード（画像 + `insertion_map.json`）

---

## 9. スコープ外（拡張フェーズ）

| 除外項目 | 理由 |
|----------|------|
| URL からブログ取得 | 拡張機能フェーズへ |
| ブラウザ拡張機能 | 同上 |
| WordPress / Ghost 等 CMS 連携 | 拡張フェーズへ |
| ユーザー認証・保存機能 | MVP 後に追加 |
| 複数言語同時対応 | 初期は日本語・英語のみ |
| 埋め込み済み Markdown / HTML 出力 | MVP 後に追加（Q3 で除外確定） |
| テーマのカスタマイズ（色変更等） | MVP 後に追加 |
